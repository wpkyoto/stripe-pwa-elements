Example3: Payment Element Usage

Documentation

What is Payment Element?

Payment Element is Stripe's unified payment form component that automatically displays relevant payment methods for your customers. It supports:

  • Credit and debit cards
  • Digital wallets (Apple Pay, Google Pay)
  • Local payment methods (based on customer location)
  • Buy now, pay later options

Advantages over Card Element:

  • Single component for all payment methods (simpler integration)
  • Automatically adapts to customer location and currency
  • Built-in support for dynamic payment methods
  • Improved conversion rates with optimized UX
  • Easier to maintain and update

Features of this example:

  • Declarative HTML approach with minimal JavaScript
  • Uses Payment Element for unified payment experience
  • Supports PaymentIntent and SetupIntent
  • Custom event handling for form submission
  • Works with server-side rendering (SSR) and static sites

Implementation Pattern: 

<!-- Declare Payment Element in HTML -->
<stripe-modal open="true">
  <stripe-payment-element
    publishable-key="pk_test_xxxxx"
    intent-client-secret="pi_xxxxx_secret_xxxxx"
    should-use-default-form-submit-action="false"
  ></stripe-payment-element>
</stripe-modal>

<script>
  // Wait for component to be defined, then set up event listener
  customElements.whenDefined('stripe-payment-element')
    .then(() => {
      const element = document.querySelector('stripe-payment-element');
      element.addEventListener('defaultFormSubmitResult', async (props) => {
        const { detail } = props;
        if (detail instanceof Error) {
          console.error('Payment failed:', detail);
          element.updateProgress('failure');
        } else {
          console.log('Payment succeeded:', detail);
          element.updateProgress('success');
        }
      });
    });
</script>

HTML Structure: 

<stripe-modal open="true">
  <stripe-payment-element
    publishable-key="pk_test_xxxxx"
    intent-client-secret="pi_xxxxx_secret_xxxxx"
    should-use-default-form-submit-action="false"
  ></stripe-payment-element>
</stripe-modal>

Attribute explanation:

  • publishable-key: Stripe publishable key (starts with "pk_test_" for test environment)
  • intent-client-secret: PaymentIntent or SetupIntent client secret (created on server-side)
  • should-use-default-form-submit-action: Whether to use default submit handling (true by default)
  • intent-type: "payment" (default) or "setup" for saving payment methods

Note: This example uses hardcoded values. In real applications, you should dynamically set values from the server-side.

Server-side setup (Node.js example): 

// Server-side code to create PaymentIntent
const stripe = require('stripe')(process.env.STRIPE_SECRET_KEY);

app.post('/create-payment-intent', async (req, res) => {
  try {
    const paymentIntent = await stripe.paymentIntents.create({
      amount: 1099, // Amount in cents
      currency: 'usd',
      automatic_payment_methods: {
        enabled: true, // Enable all available payment methods
      },
    });

    res.json({
      clientSecret: paymentIntent.client_secret,
    });
  } catch (error) {
    res.status(400).json({ error: error.message });
  }
});

Using default submit action:

If you set should-use-default-form-submit-action="true" (or omit it, as it's the default), the component will automatically call stripe.confirmPayment() when the form is submitted. 

<stripe-payment-element
  publishable-key="pk_test_xxxxx"
  intent-client-secret="pi_xxxxx_secret_xxxxx"
  should-use-default-form-submit-action="true"
></stripe-payment-element>

<script>
  customElements.whenDefined('stripe-payment-element')
    .then(() => {
      const element = document.querySelector('stripe-payment-element');

      // Listen for the result of the default submission
      element.addEventListener('defaultFormSubmitResult', async (props) => {
        const { detail } = props;
        if (detail instanceof Error) {
          // Handle error
          console.error('Payment failed:', detail.message);
        } else {
          // Handle success
          console.log('Payment succeeded:', detail);
          // Redirect to success page, close modal, etc.
        }
      });
    });
</script>

Custom submit handling: 

element.addEventListener('formSubmit', async (props) => {
  const { detail: { stripe, elements } } = props;

  // Custom payment confirmation with redirect
  const { error } = await stripe.confirmPayment({
    elements,
    confirmParams: {
      return_url: 'https://example.com/order/confirmation',
    },
  });

  if (error) {
    element.setErrorMessage(error.message);
    element.updateProgress('failure');
  } else {
    // Customer will be redirected to return_url
    element.updateProgress('success');
  }
});
